### JS 的不足

- JS 的维护成本很高，不适用于开发大型项目；

- JS 是动态类型语言，JS 里没有变量类型，函数的参数也没有类型；
- JS 不报错，会导致后面更大的错误，这点很让人头疼；

### TS 是什么？

TS 的诞生为要解决 JS 的不足，并非替换，JS 是 TS 的根，TS 是 JS  的超集。

TS 可以在任何支持 JS 的平台中执行，TS 不能被 JS 解析器直接执行，需要把 TS 编译为 JS。

TS 扩展了 JS，并添加了类型。

#### TS 增加了什么

- 类型
- 支持 ES 新特性，添加 ES 不具备的新特性
- 丰富的配置选项
- 强大的开发工具
